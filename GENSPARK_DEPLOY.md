# GenSpark Hosted Deploy - finance.gershoncrm.com

## Overview
Deploy the Gershon Finance Dashboard using **GenSpark's Hosted Deploy** feature with custom domain **finance.gershoncrm.com**.

## 🚀 Deployment Steps

### Step 1: Configure Custom Domain

**In GenSpark Deploy Tab:**
1. Navigate to **Deploy** tab in the sidebar
2. Click **Configure Custom Domain**
3. Enter domain: `finance.gershoncrm.com`
4. Follow DNS configuration instructions provided by GenSpark

### Step 2: Set Environment Variables

**In GenSpark Deploy Tab:**
1. Go to **Environment Variables** section
2. Add the following variables:

```
XERO_CLIENT_ID=<your_xero_client_id>
XERO_CLIENT_SECRET=<your_xero_client_secret>
XERO_REDIRECT_URI=https://finance.gershoncrm.com/auth/callback
```

### Step 3: Update Xero OAuth Configuration

**Update Redirect URI:**
1. Go to: https://developer.xero.com/app/manage
2. Select your OAuth app
3. Add redirect URI: `https://finance.gershoncrm.com/auth/callback`
4. Save changes

### Step 4: Deploy via GenSpark

**Option A: Automatic Deploy (Recommended)**
1. Click **Deploy** button in GenSpark Deploy tab
2. GenSpark will automatically:
   - Build the project (`npm run build`)
   - Deploy to hosting
   - Configure custom domain
   - Set up SSL certificate
   - Apply environment variables

**Option B: Manual Deploy via Git**
1. Commit all changes to git:
```bash
cd /home/user/webapp
git add .
git commit -m "Ready for production deployment"
```

2. Push to GitHub (if using GitHub integration):
```bash
git push origin main
```

3. GenSpark will auto-deploy on push

### Step 5: Verify Deployment

**Test the deployment:**
```bash
# Check health endpoint
curl https://finance.gershoncrm.com/api/health

# Expected response:
{"status":"ok","timestamp":"..."}
```

**Test in browser:**
1. Open: https://finance.gershoncrm.com
2. Should see login page
3. Click "Sign in with Xero"
4. Complete OAuth flow
5. Dashboard should load

## 🔍 Troubleshooting

### Issue: Build Fails
**Check:**
- Build logs in GenSpark Deploy tab
- Run `npm run build` locally to test
- Check for TypeScript errors

**Solution:**
```bash
cd /home/user/webapp
npm run build
# Fix any errors shown
git commit -am "Fix build errors"
```

### Issue: OAuth Redirect Error
**Cause:** Redirect URI not configured in Xero

**Solution:**
1. Go to https://developer.xero.com/app/manage
2. Add: `https://finance.gershoncrm.com/auth/callback`
3. Save and redeploy

### Issue: Environment Variables Not Working
**Cause:** Variables not set in GenSpark

**Solution:**
1. Go to Deploy tab
2. Environment Variables section
3. Add all required variables
4. Redeploy

### Issue: Custom Domain Not Working
**Cause:** DNS not configured or propagating

**Solution:**
1. Check DNS settings in GenSpark Deploy tab
2. Verify DNS records at domain registrar
3. Wait for DNS propagation (5-30 minutes)

### Issue: Dashboard Shows But No Data
**Cause:** Xero authentication failing

**Solution:**
1. Check browser console for errors
2. Verify XERO_CLIENT_ID and XERO_CLIENT_SECRET are correct
3. Verify XERO_REDIRECT_URI matches exactly
4. Test OAuth flow again

## 📝 Important Notes

### Session Management
- Sessions are stored in memory (for now)
- After deployment restart, users need to re-authenticate
- Consider implementing persistent session storage in future

### Xero OAuth
- Tokens expire after 30 minutes
- Automatic refresh is implemented
- Refresh tokens valid for 60 days

### Security
- All communication over HTTPS
- Environment variables secured by GenSpark
- OAuth 2.0 for authentication
- No sensitive data in code
